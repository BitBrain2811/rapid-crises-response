# Database Structure

The Crisis Response System uses **MongoDB** as its database. Since MongoDB is a NoSQL database, the schema is defined at the application level using Mongoose models (located in `backend/models/`).

## Collections

### `alerts`
This collection stores all the reported crisis alerts.

#### Schema Definition
- `_id`: ObjectId (Auto-generated by MongoDB)
- `type`: String (Required) - The type of crisis (e.g., "Fire", "Flood", "Earthquake").
- `location`: String (Required) - Where the crisis is happening.
- `severity`: String (Required) - The severity level. Must be one of: `['low', 'medium', 'high', 'critical']`.
- `description`: String (Optional) - Additional details about the crisis.
- `createdAt`: Date (Default: Current timestamp) - When the alert was reported.

## Setup Instructions

1. Ensure you have MongoDB installed and running locally, or use a cloud provider like MongoDB Atlas.
2. If running locally, the default connection URI is:
   `mongodb://localhost:27017/crisis_response`
3. The database name is `crisis_response`. It will be created automatically when the first alert is inserted.
